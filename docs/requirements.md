# 要件定義書 (Requirements Definition)

## 1. プロジェクト概要
Qraft（クラフト）は、ソフトウェア開発プロジェクトにおける「品質」を統合管理するための Web アプリケーションです。強力な課題管理機能に、テスト管理と欠陥（バグ）トラッキングを密に連携させることで、開発から検証までの透明性を最大化します。

## 2. 背景・目的
- **課題**: 開発の進捗（チケット）と、テストの結果（エビデンス）が別々のツールで管理され、最終的な品質判断が困難。
- **解決策**: 課題・テスト項目・欠陥を 1 つのプラットフォームで紐付け、ダッシュボードで現状をリアルタイムに見える化する。

## 3. 主要機能要件

### 3.1 課題管理 (Issue Tracking)
- [ ] 開発タスク、ユーザーストーリーをカード形式（ボード）で管理。
- [ ] ステータス遷移（Todo / InProgress / Done / Blocked）。
- [ ] 優先度と担当者の設定。

### 3.2 テスト管理 (Test Management)
- [ ] テスト項目の作成（タイトル、手順、期待される結果）。
- [ ] テスト実行結果（Pass / Fail / NoRun）の記録。
- [ ] 失敗したテスト項目からの欠陥報告（Defect 連携）。

### 3.3 欠陥管理 (Defect Tracking)
- [ ] 発見されたバグの報告と詳細記録。
- [ ] 担当者の割り当てと修正進捗の管理。
- [ ] テスト項目との紐付けによる、どの機能で問題が起きているかの可視化。

### 3.4 ダッシュボード (Insights)
- [ ] プロジェクトの進捗率（バーンダウン）。
- [ ] テスト通過率（パスレート）の可視化。
- [ ] 累積デフェクト数のトレンドグラフ。

## 4. 非機能要件
- **パフォーマンス**: Cloudflare エッジネットワークによる、グローバルでの高速応答。
- **信頼性**: D1 データベースによるトランザクション整合性と自動バックアップ。
- **拡張性**: Hono による軽量かつスケーラブルな API 構造。

## 5. ターゲットユーザー
- プロジェクトマネージャー (PM)
- ソフトウェア開発エンジニア (Dev)
- 品質保証・テストエンジニア (QA)
