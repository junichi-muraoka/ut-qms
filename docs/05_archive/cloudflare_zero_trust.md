Qraft プロジェクトにおける認証基盤として、Cloudflare Zero Trust (Access) を採用しています。このドキュメントでは、その導入理由と仕組みについて解説します。

## 1. Cloudflare Zero Trust とは？
インターネット上の特定のアプリケーションに対して、「誰が（ID）」「どのデバイスから」アクセスできるかを細かく制御するセキュリティ基盤です。

### なぜ Qraft に導入したのか？
- **パスワード管理からの解放**: 開発者が自前で「ユーザー登録・パスワード再発行」の仕組みを作る必要がありません。
- **強力なアクセス制限**: 特定のメールアドレス（村岡様のみ、またはチーム全員など）以外からのアクセスを、ネットワークの入り口で完全に遮断します。
- **プロ仕様のUI**: ログインしたユーザーの名前やアバターを、Google アカウントなどの情報から取得して画面上に表示できます。

## 2. コスト（料金）について
Cloudflare Zero Trust には、個人開発や中小チームに最適な **Free プラン** があります。

- **ユーザー数**: **50人まで 0円**
- **通信量・ログイン回数**: 無制限で 0円
- **自動課金の心配**: Free プランを選択している限り、勝手にお金がかかることはありません。

## 3. Qraft での仕組み
システム的には、以下の流れで「安全なログイン」を実現しています。

1.  **Cloudflare Access**: ユーザーがサイトにアクセスした際、Cloudflare が認証（OTPやGoogle等）を行い、「JWT (JSON Web Token)」という証明書を発行します。
2.  **Backend (Workers)**: 送られてきた証明書が本物かどうかを検証し、正しければユーザー情報（email等）を取得します。
3.  **Frontend (React)**: バックエンドから受け取った情報を元に、ヘッダーにユーザーのアイコンを表示します。

## 4. 今後の拡張性
この基盤があることで、将来的に以下の機能を簡単に追加できます。
- 「このタスクの担当者を A さんにする」といったユーザー紐づけ
- 特定の操作（削除など）を「管理者のみ」に制限する権限管理
